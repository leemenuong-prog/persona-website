Why the current AIPM loses to the Developer

→ AIPM 改为「数据自我变形」

Content

两者都终结于同一个物体（3D 的 .IAM. 天际线）、同一个手势（搭起来）——这就是「太像」的根源。差异化的杠杆是运动性格：Developer 是你旋转的实体；AIPM 应是你看它思考的过程。

A

遥测之雨 · Telemetry Rain

首席推荐

噪声落雨，自行归位，沉成那个标记——然后标记立起，长成维度。 · Noise rains, sorts, settles into the mark — then the mark stands up into dimension.

一场约 1,800 点的遥测之雨落下、自行归入八根直方柱——而这八根柱正是 .IAM. 的节奏；扁平的柱子随即从图表平面立起为三维维度棱柱，蓝色句点落定。数据先变成品牌，再变成空间。

★ Wow beat

pc≈0.40：你盯了半天当作噪声的雨，突然归位、分箱、堆叠，活跳的抖动在八根柱锁定到 .IAM. 节奏的同一帧里死去——剪影瞬间就是那个标记。是「认出」，不是「揭晓」：数据一直在拼这个 logo。

与 Developer 的区别

开场是 1,800 个抖动的点（实心方块的反面），全程不可拖拽（导演运镜），天际线只占最后 30%——前提是「排序」不是「搭建」。两者只在最后共享 .IAM. 剪影，却由相反路径抵达。

三维 · 多维度

八根扁平柱同时挤出深度成棱柱，相机从正视图抬到轴测；后排棱柱以航空雾向钴蓝褪去——「多个具象维度」用空气透视表达，靠明度不靠颜色。

可行性 · 最高

复用现有 project() 相机、pushBox、画家排序、settleA 落点闪光。点全部由 pc 纯函数定位、抖动只是 now 装饰 → 反向滚动稳定。1,800 个 arc 在收束后并入 8 个盒子，填充率持平。

B

结晶 · Crystallisation

万千遥测寻得一根真轴，坍折成一枚澄澈的晶。 · A storm of measurements finds one true axis, then folds into a single clear gem.

抖动的三维遥测群被一根回归主轴贯穿（流星式亮头绘出），随后把自身的点折叠成一枚低多边形晶体的棱面；晶体最终解理、重排成 .IAM. 天际线。分析即结晶，而非搭建——全站最像「珍宝」的一帧。

★ Wow beat

pc≈0.36–0.50：安静的点云在两三帧内「折」成一枚实心晶体——你当作噪声盯着的同一批点，正是这枚宝石的顶点，棱面在它们之间嵌入。混沌没有被替换，而是从溶液里析出。

与 Developer 的区别

主角历经四态（云→纺锤→晶→天际线），晶体/宝石这一形态全站独一无二，没人会把它和天际线搞混；相机是不可拖拽的导演式缓推。

三维 · 多维度

每个棱面 = 一个维度（研究建议 5–8 面最易读，正好 8）。深度全靠平面着色的高光边，钴蓝只落在棱线与顶点，绝不整面铺色——符合极简单色 3D 的「贵」。

可行性 · 中（最高回报）

约 240 点 + 16 棱面 + 8 柱 ≈ 300 面/帧，比现 genesis 还轻。风险只在「解理」：柱必须由晶体自己的棱面簇连续形变而来，不能淡入淡出——否则又像 Developer 的搭建。

C

解码 · Decode（字⇄条）

每一个数字本来就是一栋楼，分析不过是学会去读它。 · Every number was already a building; analysis is just learning to read it.

一片原始等宽数字像活遥测般抖动，在扫描读头下归位，沿本站自有的「字⇄条」语法折成图表——图表随即从纸面升起为一片维度高塔，相机直接穿行而过（透视，非轴测），最后退场留下 .IAM. 与蓝色句点。这是把品牌语法本身演成动画。

★ Wow beat

pc 0.30–0.50：一列列看似随机的遥测数字「折」成柱子，且抖动在同一刻死去——你刚才在读的数字串，原来就是 .IAM. 的节奏。「读」即「解码」，把本站的 字⇄条 语法字面执行。

与 Developer 的区别

开场是纯排版（一团 2D 文字），全站独有；唯一使用透视短焦让相机穿过塔林（消失点、视差冲刷）——正是 Developer 平行轴测正交的反面。

三维 · 多维度

主柱之外，暗弱的「被否决的假设」高塔从画面边缘掠过被甩在身后——「提炼」在视觉上就是穿过许多维度只留下站得住的那几根。

可行性 · 中

复用全部 3D 基元；新成本是约 220 个 fillText（封顶、远处剔除、morph>0.6 后停止绘制）。风险是品味而非性能：文字易「Matrix 感」——靠大量留白、单一读头、一次硬切 morph 化解。

References — what I looked at first
  • Bruno Simon — 整站就是一个「单一英雄时刻」，把导航变成可玩的物件
  • Active Theory — 电影感而非装饰；滚动是穿越 3D 环境
  • Aristide Benoist — 克制即对比：九成安静，一记缓动转场说全部
  • The Pudding · 散点→蜂群→面积图 — 对象恒常性：同一个点从散点走到柱
  • Apple AirPods 滚动序列 — 滚动即时间轴，可逆、可端详
  • Particles Morphing — 粒子在两套目标间插值 = 干净的「群→标记」
  • bit-101 透视数学 · 画家算法 — 纯 2D canvas 假 3D 的全部把戏
  • 雷达图 / 低多边形晶体 — 每个面 = 一个维度，5–8 面最易认

提炼出的两条最重要原则：① 对象恒常性——同一个点从散点移动到柱再到 3D 顶点（重生=装饰，重定位=推理）。② 高潮要「凝聚」不要「四散」——结晶之所以读作「洞见」，正因粒子向内汇聚。两条都验证了上面三稿的脊柱。

首席设计师的判断

三稿共用同一条正确的脊柱：开场是被读作噪声的流动之物 → 在秩序到来的同一帧里杀死抖动 → 把 .IAM. 作为「认出」而非「揭晓」浮现 → 蓝色只触碰句点。它们都复用 genesis 的相机与 pushBox/雾/settleA 基元，构建成本有界，且因布局是 pc 的纯函数而可反向滚动。

你只需要做一个气质判断，其余（数字、点数、穿行 vs 轴测）都是下游：

必然感 · Inevitable

A 雨 / C 解码 —— 系统把噪声收敛成秩序，品牌作为显然的答案浮现。最易读、最低风险、单点 aha 最强。「秩序即产品」最干净的陈述。

珍宝感 · Precious

B 结晶 —— 品牌作为从溶液析出的多面宝石，全站最美的物件，但最需要克制的导演，否则会滑向「好看的屏保」而丢掉认知论证。

我的票：A 遥测之雨。它是「Order is the Product」最干净的视觉陈述、构建风险最低、单点 aha 最强；若日后想要更多奇观，B 的「结晶」折面节拍可作为可选中段华彩嫁接进来，不动脊柱。告诉我选哪个方向，我就把它实装进 aipm-genesis.jsx。
